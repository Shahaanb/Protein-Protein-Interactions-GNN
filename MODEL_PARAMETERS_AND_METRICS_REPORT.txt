0) Project Features (what the demo does)
---------------------------------------

User workflow
- Enter two PDB IDs (receptor + ligand).
- Run a single inference pass.
- View:
  - Interaction Probability (model output mapped to a probability-like value)
  - Evidence Score (robustness + sanity heuristic; NOT model self-confidence)
  - Hotspots (top residues by an attention-like signal)
  - 3D structures with hotspot residues highlighted.

Interpretability / transparency
- Hotspots are an interpretability signal (“where the model focused”), not a probability.
- Evidence Score returns an explicit breakdown so users can see what contributed.
- Model Scores modal shows benchmark-style metrics and 95% bootstrap CIs from a lightweight DB5.5 pass.


1) Inference API (FastAPI)
-------------------------

1.1 Endpoint: POST /predict
Request body (JSON)
- pdb1 (string, required)
  - First protein PDB ID (expected 4-character code, case-insensitive).
- pdb2 (string, required)
  - Second protein PDB ID (expected 4-character code, case-insensitive).

Response body (JSON)
Top-level
- source ("cache" | "compute")
  - Indicates whether the response came from Redis cache or fresh inference.
- data (object)
  - The prediction payload (see below).

Prediction payload fields (data)
- pdb_pair (string)
  - Format: "{PDB1}_{PDB2}" uppercased.
- pdb1 (string)
- pdb2 (string)

- interaction_probability (number)
  - Units: percent (%), range 0–100.
  - Meaning: the model’s first ONNX output mapped to a probability-like value and returned as a percentage.
  - Computation:
    - Read ONNX output[0] as raw_prob.
    - If raw_prob is already in [0,1], treat it as prob.
    - Otherwise treat it as a logit and convert using sigmoid:
        prob = 1 / (1 + exp(-raw_prob))
    - Returned as: round(prob * 100, 2)
  - Caveat: this is not guaranteed to be calibrated to real-world interaction frequencies.

- inference_mode ("onnx" | "mock")
  - "onnx": ONNXRuntime successfully loaded the model and ran inference.
  - "mock": deterministic mock output (used if model load/inference or PDB fetch/parse fails).

- hotspots (array of objects)
  - Purpose: interpretability UI signal, not a probability.
  - Each hotspot object:
    - node_idx (integer)
      - In ONNX mode: residue number derived from PDB CA records.
      - In mock mode: plausible synthetic residue index.
    - attention (number)
      - Attention-like per-node scalar extracted from model outputs (or generated deterministically in mock mode).
  - Selection: top-5 residues by descending attention.

- evidence_score (number)
  - Scale: 0–100.
  - Meaning: robustness + input sanity heuristic (explicitly NOT the model’s self-confidence).

- evidence_breakdown (object)
  - Meaning: structured explanation of evidence_score.
  - Fields:
    - metric_version: "evidence_v1"
    - inference_mode: "onnx" or "mock"
    - runs: total evidence runs (see EVIDENCE_RUNS)
    - feature_noise_std: feature noise std for additional runs
    - prob_std_max_percent_points: scaling constant for probability stability
    - weights: weights for sub-scores (probability_stability, hotspot_consistency, input_quality)
    - probability_std_percent_points: std-dev of interaction_probability over runs (percent points)
    - probability_stability_score: 1 - min(1, prob_std / EVIDENCE_PROB_STD_MAX)
    - hotspot_consistency_score: mean pairwise Jaccard similarity of hotspot sets over runs
    - input_quality_score: heuristic from graph node/edge counts
    - mode_factor: down-weight in mock mode
    - notes: human-readable caveat

2) Inference Environment Parameters (backend/inference.py)
----------------------------------------------------------

All are optional environment variables; defaults shown.

Model / runtime
- MODEL_PATH (default: "proteome_x_gat.onnx")
  - Path to ONNX model file (relative to backend working directory).

- ORT_LOG_SEVERITY_LEVEL (default: 3)
  - 0=verbose, 1=info, 2=warning, 3=error, 4=fatal.

Graph construction
- GRAPH_DISTANCE_THRESHOLD (default: 8.0)
  - Units: Å (Angstroms) for CA–CA distance cutoff.
  - Meaning: directed edges are created between any two nodes with distance < threshold.

- GRAPH_FEATURE_DIM (default: 64)
  - Node feature vector dimensionality.

Evidence Score parameters
- EVIDENCE_RUNS (default: 3)
  - Total runs used to estimate stability.
- EVIDENCE_FEATURE_NOISE_STD (default: 0.005)
  - Gaussian noise std added to node features for stability checking.
- EVIDENCE_PROB_STD_MAX (default: 8.0)
  - Units: percent points (since stability is computed on prob*100).
- EVIDENCE_MODE_FACTOR_ONNX (default: 1.0)
- EVIDENCE_MODE_FACTOR_MOCK (default: 0.15)
  - Final evidence multiplier depending on inference_mode.

PDB download
- Source: https://files.rcsb.org/download/{PDB}.pdb
- Fetch timeout: 10s
- Cached in-process via LRU cache.


3) Cache Parameters (backend/cache.py)
-------------------------------------

Environment variables

4) Validation / Benchmark-Style Metrics
--------------------------------------

4.1 CLI: backend/run_validation.py
Purpose
- Runs a lightweight DB5.5 evaluation pass and writes:
  - backend/validation/outputs/validation_results.json (full)
  - backend/validation/outputs/validation_summary.json (slim, UI-friendly)

Arguments (defaults)
- --max-positives (int, default 40)
  - Max DB5.5 positive pairs to score.
- --negatives-per-positive (int, default 1)
  - Synthetic negatives per positive (mismatched ligands).
- --n-boot (int, default 200)
  - Bootstrap resamples for 95% CIs.
- --seed (int, default 1337)
  - RNG seed for negative sampling and bootstrap.

Split strategy
- Leakage-safe split: group split by receptor (pdb1)
- Fractions: train=0.70, val=0.15, test=0.15

Threshold selection
- If validation split has >= 20 examples: choose threshold maximizing F1 on val.
- Otherwise: fixed threshold = 0.5.


4.2 Metrics computed on the TEST split (definitions)
All metrics are computed from:
- y_true: ground truth labels (1=positive, 0=negative)
- y_prob: predicted probabilities in [0,1]
- y_pred: (y_prob >= threshold)

Confusion counts
- tp, tn, fp, fn

Classification metrics
- accuracy = (tp + tn) / (tp + tn + fp + fn)
- precision = tp / (tp + fp)
- recall = tp / (tp + fn)
- f1 = 2 * precision * recall / (precision + recall)

Ranking metrics
- roc_auc: area under ROC curve; 0.5 ≈ random ranking.
- pr_auc: area under PR curve; baseline depends on positive prevalence.

Calibration metrics
- brier = mean( (y_prob - y_true)^2 )
- ece_10 (expected calibration error, 10 equal-width bins)
  - For each bin: compare mean predicted probability vs mean true label.
  - ece = sum_bin( bin_weight * |acc_bin - prob_bin| )

4.4 Uncertainty: Bootstrap 95% confidence intervals
- test_metrics_ci95 provides CIs for:
  accuracy, precision, recall, f1, roc_auc, pr_auc, brier, ece_10

How the CI is computed
- Stratified bootstrap:
  - Sample positives with replacement (same count as original positives).
  - Sample negatives with replacement (same count as original negatives).
  - Combine and shuffle.
  - Recompute metrics.
- Repeat n_boot times.
- 95% CI = [2.5th percentile, 97.5th percentile] of the bootstrap metric distribution.

Important caveat
- With very small test splits, CIs can be wide or unstable.
- If the test split contains a single class (all 0s or all 1s), CI computation returns an empty object.

6) Scientific & Demo Caveats (Read This)
----------------------------------------

This validation is intentionally lightweight and demo-oriented:
- Negatives are synthetic mismatches, not curated non-interacting pairs.
- Chain-level identities in DB5.5 table are ignored (only the first 4 chars are used).
- The model is not re-trained here; this evaluates the current inference output “as-is.”
- “Interaction Probability” is a model score; it may not be calibrated.
- “Evidence Score” measures robustness/sanity, not correctness.

- Increase validation sample sizes (e.g., max_positives=150+) for more stable metrics.
- Add a calibration plot artifact (reliability curve) to the validation outputs.
- Add per-split AUC/PR-AUC and stratified reporting.
